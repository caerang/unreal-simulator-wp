# ADR 0001: 언리얼 엔진 gRPC 클라이언트 구현 방식 결정

- **상태 (Status):** 승인됨 (Accepted)
- **작성일 (Date):** 2026-06-13
- **결정자 (Deciders):** Antigravity (AI Agent), User

---

## 1. 맥락 (Context)
언리얼 엔진 5 기반의 임무 비행 시뮬레이터(클라이언트)와 파이썬 기반의 강화학습 에이전트(서버) 간에 매 Tick마다 상태 데이터(State)와 행동 데이터(Action)를 주고받을 통신 채널이 필요합니다. 
현재 언리얼 엔진 프로젝트는 C++ 소스 코드가 없는 순수 블루프린트 프로젝트로 구성되어 있으며, 고속/대규모(256개+ 병렬 환경) 강화학습을 지원하기 위해 gRPC 클라이언트를 연동하는 방안을 모색해야 합니다.

## 2. 대안 및 비교 (Alternatives Considered)

### 대안 1: C++ 직접 빌드 및 바인딩 (Manual C++ Integration)
- **설명:** 언리얼 프로젝트를 C++로 전환하고 gRPC 공식 C++ SDK와 Protobuf 소스 및 종속 라이브러리(OpenSSL, zlib 등)를 직접 Windows/MSVC 환경에서 빌드하여 언리얼 빌드 시스템(UBT)에 연결합니다.
- **장점:** 성능이 최상이며 gRPC 사양을 100% 제어 가능합니다.
- **단점:** Windows 환경에서 외부 C++ 라이브러리 정적/동적 링크 및 UBT 충돌을 해결하기 위한 초기 설정 난이도가 매우 높고 빌드가 자주 깨집니다.

### 대안 2: 언리얼 엔진용 오픈소스 gRPC 플러그인 도입 (TurboLink 등)
- **설명:** 언리얼 엔진 환경에 맞춰 사전 빌드된 gRPC 바이너리와 `.proto` 파일을 기반으로 C++ 바인딩 코드 및 블루프린트 노드를 자동 생성해 주는 오픈소스 플러그인을 도입합니다.
- **장점:** 빌드 환경 구성의 치명적인 복잡도가 해결되어 있고, 스키마 컴파일 시 블루프린트 연동 노드가 자동 생성되어 개발 속도가 매우 빠릅니다. 바이너리(Protobuf) 통신을 수행하므로 성능 또한 우수합니다.
- **단점:** 엔진 버전이 대폭 업데이트될 경우 플러그인 업그레이드 대응이 필요합니다.

### 대안 3: 파이썬 중개 게이트웨이(WebSocket/HTTP) 경유
- **설명:** 파이썬 측에 웹소켓/HTTP 통신을 수신하여 gRPC 서버로 중개하는 게이트웨이를 띄우고, 언리얼은 엔진 기본 웹소켓/HTTP 블루프린트 노드로 통신합니다.
- **장점:** C++ 컴파일 없이 순수 블루프린트로 즉시 연동되어 빌드 안정성이 가장 높습니다.
- **단점:** 대규모 병렬(256개+ 에이전트) 고속 루프 구동 시 JSON 직렬화/역직렬화 오버헤드가 누적되어 극심한 CPU 성능 병목 및 통신 지연이 발생합니다.

## 3. 결정 사항 (Decision)
**대안 2 (언리얼 gRPC 전용 플러그인 - TurboLink 등 도입)** 방식으로 진행합니다.

**선택 사유:**
1. **성능 요구사항 충족:** 256개 이상의 다중 에이전트 학습 시 매 Tick마다 바이너리 직렬화가 필수적이므로 방식 3(웹소켓 JSON)은 성능 한계로 배제합니다.
2. **생산성 및 개발 안정성:** 직접 gRPC C++ 라이브러리를 바인딩하는 방식 1은 빌드 환경 셋업 오버헤드가 과도하므로, 검증된 오픈소스 플러그인을 도입하여 효율적이고 안전하게 연동합니다.

## 4. 파급 효과 (Consequences)

### 긍정적 효과
- `.proto` 스키마 수정 시 블루프린트 및 C++ 연동 노드가 자동 컴파일되어 개발 생산성이 극대화됩니다.
- 바이너리 프로토콜 활용으로 네트워크 대역폭 및 언리얼/파이썬 프로세스 CPU 부하를 최소화합니다.

### 조치 필요 사항 (Next Action Items)
1. 언리얼 엔진 프로젝트 디렉토리에 `Plugins/` 폴더를 생성하고, 언리얼 엔진 버전에 맞는 **TurboLink** (또는 호환 gRPC 플러그인)을 임포트합니다.
2. 컴파일을 트리거하기 위해 프로젝트를 C++가 포함된 모듈 형식으로 빌드할 수 있도록 빈 C++ 클래스 생성 및 프로젝트 C++ 전환(Visual Studio 솔루션 생성)을 수행합니다.
3. `.proto` 컴파일 스크립트 작성 및 플러그인을 활용한 블루프린트 통신 구현을 이어서 진행합니다.
