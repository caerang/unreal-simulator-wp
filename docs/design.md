# 프로젝트 설계 사양서 (Design Specification)

본 문서는 **Unreal Engine 임무 비행 시뮬레이터 & gRPC 강화학습 에이전트 파이프라인 연동 프로젝트**의 상위 설계 사양을 정의합니다.

> 상세 아키텍처 구현 이력은 [UE5_MultiAgent_Simulator_Architecture.md](file:///D:/workspace/simulator_wp/docs/UE5_MultiAgent_Simulator_Architecture.md)를 참고합니다.

## 1. 시뮬레이터

- 언리얼 엔진 5(UE5)를 사용해서 구현한다.
- 시뮬레이터는 임무 비행 시나리오에 따라 전투기가 작전을 수행한다.
- 임무 비행 시나리오는 행위 트리(Behavior Tree) 기반으로 구현한다.
- C2(Commander&Control)는 임무 비행 시나리오에서 어떤 단계를 수행해야 하는지 결정하고 현재 수행해야 하는 단계를 지시한다.
- 임무 수행 전투기는 임무 수행 단계에 따라 수행해야 하는 행위 트리에 따라 세부 작전(기동)을 수행한다.
- 다양한 임무 비행 시나리오가 생성 될 수 있으며, 동일한 임무에 대해서 시작 위치, 타격 대상 종류, 타격 대상 위치 등 다양한 임무 환경(상황)을 설정하고 변경할 수 있다.
- 임무 수행 단계 결정은 현지 작전 상황, 비행기 상황 등에 따라 규칙 기반 알고리듬에 따라 판단한다.

### 1.1. 데이터 주도형 시나리오 관리
- 시나리오 데이터는 외부 CSV 파일(`Scenario_List.csv`)로 관리하며, 언리얼 엔진 데이터 테이블로 임포트하여 사용한다.
- 시나리오별 매개변수: 시나리오 ID, 타겟 종류(Drone/Jet/Ship), 스폰 위치(X/Y/Z) 등.

### 1.2. AI 제어 구조
- 전투기는 `Pawn` 클래스 기반이며, AI 컨트롤러(`AIC_Fighter`)가 자동 빙의(Auto Possess AI)하여 제어한다.
- 블랙보드(`BB_Fighter`)에 임무 데이터를 기록하고, 행위 트리(`BT_Fighter`)가 이를 기반으로 행동을 분기한다.
- 초기화 순서: `OnPossess` -> `Use Blackboard` -> `Set Value` -> `Run Behavior Tree` (명시적 순서 보장).

## 2. 임무 비행 에이전트

- 강화학습 기반으로 수행 단계에 따라 세부 작전 수행 모델을 학습한다.
- 강화학습 기반으로 세부 작전을 수행하는 모델은 규칙 기반 알고리듬을 따르는 것이 아니고 학습에 사용한 데이터에 따라 최적의 세부 작전 기동을 학습한다.
- 파이썬 기반으로 구현하며, PyTorch, RLlib 등 프레임워크를 활용한다.

---

## 3. 통신 파이프라인 (gRPC)

- 언리얼로 구현한 시뮬레이터와 파이썬으로 구현한 임무 비행 에이전트가 **gRPC**를 사용해서 강화학습에 필요한 상태 데이터와 행동 데이터를 서로 주고 받는 파이프라인을 구축한다.
- **Python(에이전트) = gRPC Server**, **UE5(시뮬레이터) = gRPC Client** 구조로 배치한다.
  - UE5가 매 Tick마다 State를 Python 서버로 전송하고, Python이 Action을 응답하는 방식.
  - 다수 환경의 State를 배치(Batch)로 모아 GPU에서 한 번에 추론하여 학습 효율을 극대화한다.
  - 상세 분석: [grpc_server_placement_analysis.md](file:///D:/workspace/simulator_wp/docs/grpc_server_placement_analysis.md)
- gRPC 채택 사유: 클라우드 GPU 서버와의 원격 분산 학습, 파이썬 프레임워크 호환성, 256개+ 스케일업 시 서버/클라이언트 아키텍처 분리 용이성.

### 3.1. 프로토콜 버퍼 메시지 구조
- **AgentState (UE -> Python):** 환경 ID, 에이전트 종류, 3차원 좌표(X/Y/Z), 방향, 타겟 거리 등 관측 정보.
- **AgentAction (Python -> UE):** 엔진 추진력(Thrust), 조향(Steering) 등 파이썬이 계산한 제어 수치.

## 4. 대규모 병렬 실행 (Multi-Scenario)

- 임무 시나리오는 다수개(32개, 64개, 최대 256개+)의 시나리오가 동시에 실행 가능해야 한다.
- 각각의 시나리오가 실행 될 때, 임무 비행 에이전트는 각각의 개별 시나리오와 파이프라인을 통해 상태 데이터와 행동 데이터를 주고 받을 수 있어야 한다.

### 4.1. 공간 분할 병렬 처리 아키텍처
- **지형 기반 오프셋(Geographic Offset):** 시나리오별 거점의 절대 좌표를 중심으로 에이전트를 스폰하여 논리적/물리적 환경을 격리한다.
- **LWC 및 단일 틱 공유:** UE5의 거대 월드 좌표(Large World Coordinates)를 활용하여 좌표 오차를 제거하고, 모든 환경이 동일한 Time Step을 공유한다.
- **헤드리스 모드:** 학습 시에는 `-nullrhi` 커맨드를 활용한 헤드리스 모드로 구동하여 GPU 렌더링 부하를 제거한다.
